# Risks And Trade-Offs

- GitHub Pages is simple and free, but cannot receive webhooks.
- Serverless webhooks are low-cost, but add a separate runtime from the Python research code.
- GitHub repository logs are easy to inspect, but are not a good high-volume event database.
- Public APIs reduce cost, but may have rate limits, stale data, or inconsistent symbols.
- Telegram alerts are effective, but too many Level 2 alerts can create fatigue.
- LLM analysis is useful for synthesis, but creates cost and hallucination risk.
- Paper trading helps validate process, but does not prove live execution quality.
